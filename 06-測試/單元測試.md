# 單元測試

> 單元測試的目標是驗證最小功能單位是否正確，例如函式、hook、reducer、utils。

> [[前端工程師]]

---

## 適合測什麼

- utils function
- formatter
- reducer
- 自訂 hook
- 純邏輯函式

---

## 為什麼重要

因為這些地方最容易：

- 改壞
- 有邊界條件
- 被多處重用

---

## Vitest 範例

```ts
import { describe, it, expect } from 'vitest';
import { formatPrice } from './formatPrice';

describe('formatPrice', () => {
  it('formats number correctly', () => {
    expect(formatPrice(1000)).toBe('$1,000');
  });
});
```

---

## 好的單元測試特徵

- 測單一行為
- 名稱清楚
- 不依賴太多外部環境
- 執行快

---

## 常見誤區

### 什麼都 mock

會讓測試變成只是在測 mock。

### UI 細節全塞進單元測試

這常常會讓測試很脆弱，很多 UI 流程更適合 integration 或 E2E。
