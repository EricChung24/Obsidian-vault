# 渲染優化

> 渲染優化的核心不是讓畫面「完全不 render」，而是降低不必要的重算與大範圍更新。

> [[前端工程師]]

---

## Virtual DOM

React 不是每次 render 都把整個 DOM 全部重建。

它的流程更接近：

1. 重新計算新的 UI 結構
2. 比對前後差異
3. 只更新真正變動的 DOM

---

## 常見優化手法

### 1. 元件切分

不要讓高頻更新 state 放在太高層，否則整棵樹都容易一起更新。

### 2. `React.memo`

適合擋住 props 沒變時的不必要 render。

### 3. `useMemo`

適合快取昂貴計算，不要什麼都包。

### 4. `useCallback`

適合需要穩定函式參考的場景，例如搭配 memo 子元件。

---

## 列表優化

大列表是最常出現效能瓶頸的地方。

你可以考慮：

- 分頁
- Infinite Scroll
- 虛擬列表

如果一次渲染幾百筆以上資料，通常就值得開始想這件事。

---

## debounce 與 throttle

### debounce

適合：

- 搜尋輸入
- resize
- 避免短時間重複請求

### throttle

適合：

- scroll
- mousemove
- 高頻事件限制觸發頻率

---

## 常見誤區

### render 就等於效能差

不對。問題是「不必要的大量 render」，不是 render 本身。

### 所有元件都該 memo

不對。很多小元件根本不需要。
