# 前端資安

> 前端資安不是只靠後端處理。前端工程師常會直接接觸輸入、token、cookie、第三方腳本與瀏覽器 API，所以一定要懂基本攻擊面。

> [[前端工程師]]

---

## XSS

XSS 是最常見的前端攻擊之一，重點是攻擊者把惡意腳本注入頁面中執行。

### 危險寫法

```js
element.innerHTML = userInput;
```

```jsx
<div dangerouslySetInnerHTML={{ __html: userInput }} />
```

### 較安全寫法

```js
element.textContent = userInput;
```

```jsx
<div>{userInput}</div>
```

如果真的要渲染 HTML，應搭配 sanitization 工具。

---

## CSRF

CSRF 是利用使用者已登入狀態，誘導他送出非本人意圖的請求。

這在 cookie-based authentication 特別常見。

常見防護：

- SameSite Cookie
- CSRF token
- 關鍵操作二次確認

---

## CSP

CSP 是 Content Security Policy，用來限制哪些來源可以載入腳本、樣式、圖片等資源。

```http
Content-Security-Policy: default-src 'self';
```

它對降低 XSS 風險很有幫助。

---

## 常見前端資安重點

- 不要信任使用者輸入
- 不要把敏感資料亂存在可讀 storage
- 第三方 script 要控管來源
- token / cookie 的使用要理解風險

---

## 常見誤區

### React 會自動幫你處理所有 XSS

React 對一般字串插值有保護，但你只要使用 `dangerouslySetInnerHTML`，風險就回來了。

### token 放 localStorage 一定沒問題

不一定，這跟你的整體安全設計有關。
