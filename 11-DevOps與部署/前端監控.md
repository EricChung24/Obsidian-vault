# 前端監控

> 前端監控的目的是在使用者真正遇到問題時，你能知道發生了什麼，而不是只靠使用者截圖回報。

> [[前端工程師]]

---

## 前端監控在看什麼

通常會分成兩大類：

- 錯誤監控
- 效能監控

---

## 錯誤監控

常見工具像 Sentry。

你會想知道：

- 哪個頁面噴錯
- 哪種錯誤最常發生
- 哪些使用者受影響
- 哪次版本開始出問題

---

## 效能監控

除了 JS error，也要看：

- 頁面載入時間
- API 速度
- Core Web Vitals
- 使用者互動延遲

---

## 常見接法

### 初始化工具

例如在專案入口初始化：

- DSN
- environment
- release version

### 標記上下文

像是：

- 使用者 ID
- 當前頁面
- 操作行為

這樣 error 才有分析價值。

---

## 為什麼它重要

因為很多 bug：

- 你本地不會發生
- 測試環境不一定重現
- 只在特定瀏覽器或網路條件出現

沒有監控，你只能猜。

---

## 常見誤區

### 只看 console error 就夠

不夠。使用者現場情境通常拿不到完整資訊。

### 有裝監控工具就算做完

不對。你還要設：

- 版本資訊
- ignore 規則
- 告警流程
