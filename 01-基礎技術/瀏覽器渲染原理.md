# 瀏覽器渲染原理

> 了解瀏覽器如何把 HTML/CSS/JS 變成你看到的畫面。

← [[前端工程師]]

---

## 關鍵渲染路徑

```
HTML → DOM Tree ─┐
                 ├→ Render Tree → Layout → Paint → Composite
CSS  → CSSOM   ─┘

JS 可阻塞 HTML 解析（除非 defer/async）
```

---

## 渲染流水線

### 1. 解析 HTML → DOM

```
<html>           Document
  <body>    →      └─ html
    <h1>              └─ body
      Hello               └─ h1
                              └─ "Hello"
```

### 2. 解析 CSS → CSSOM

所有 CSS 都要解析完才能建構 CSSOM（阻塞渲染）。

### 3. 合成 Render Tree

只包含**可見**節點（`display: none` 不在 Render Tree 中）。

### 4. Layout（重排 Reflow）

計算每個節點的**精確位置和尺寸**，耗費高。

**觸發 Reflow 的操作：**
- 增刪/修改 DOM
- 修改 width/height/padding/margin
- 讀取 `offsetWidth`、`getBoundingClientRect()`

### 5. Paint（重繪 Repaint）

將節點轉換成實際像素，耗費中等。

**只觸發 Repaint（不觸發 Reflow）：**
- 修改 color、background-color、box-shadow

### 6. Composite（合成）

最便宜的操作，**僅觸發合成**：
- `transform`
- `opacity`
- `filter`（GPU 加速）

## 效能優化建議

```css
/* 用 transform 代替 top/left（避免 Reflow）*/
.bad  { left: 100px; }     /* 觸發 Reflow */
.good { transform: translateX(100px); } /* 只有 Composite */

/* 提升為合成層（謹慎使用）*/
.gpu-accelerated {
  will-change: transform;  /* 提示瀏覽器預先建立圖層 */
  transform: translateZ(0); /* 舊版 hack */
}
```

```js
// 批次 DOM 讀寫（避免強制同步佈局）
// 錯誤：讀寫交替觸發多次 Reflow
element1.style.width = element2.offsetWidth + 'px';
element2.style.width = element3.offsetWidth + 'px';

// 正確：先全部讀，再全部寫
const w2 = element2.offsetWidth;
const w3 = element3.offsetWidth;
element1.style.width = w2 + 'px';
element2.style.width = w3 + 'px';
```

## requestAnimationFrame

```js
// 用於動畫 — 在瀏覽器下一次重繪前執行
function animate(timestamp) {
  const progress = (timestamp - start) / duration;
  element.style.transform = `translateX(${progress * 300}px)`;

  if (progress < 1) {
    requestAnimationFrame(animate);
  }
}

requestAnimationFrame(animate);
```
