# 前端系統設計回答模板

> 系統設計面試最怕想到哪講到哪。這篇的目的是給你一個固定框架，讓回答更穩。

## 回答框架

1. 需求與範圍
2. 頁面與模組拆分
3. 資料流與狀態管理
4. 效能與錯誤處理
5. 監控與部署

---

## 範例模板

> 我會先確認需求，例如使用者角色、核心流程、資料量與即時性。接著我會拆頁面模組，例如列表區、詳情區、操作區，再決定哪些 state 是局部、哪些是全域、哪些屬於 server state。之後再補 API layer、錯誤處理、效能策略，最後補監控與部署流程。

---

## 題型套用

### Feed / Dashboard

重點講：

- 分頁與快取
- 元件拆分
- 狀態管理

### 表單系統

重點講：

- 欄位驗證
- 狀態設計
- 錯誤處理

### 即時系統

重點講：

- WebSocket / polling
- 狀態同步
- 效能與監控
