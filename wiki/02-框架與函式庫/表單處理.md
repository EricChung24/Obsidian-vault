# 表單處理

> 表單是前端最常見、也最容易出錯的互動之一。重點不只是把欄位畫出來，而是處理輸入、驗證、錯誤、提交與狀態管理。

> [[前端工程師]] | [[React]] | [[TypeScript]]

---

## 手寫表單的常見問題

如果全部自己用 `useState` 管：

- 欄位很多時很容易膨脹
- 驗證邏輯分散
- error state 難維護

所以中大型表單常會考慮 React Hook Form。

---

## React Hook Form 基本用法

```tsx
import { useForm } from 'react-hook-form';

function LoginForm() {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm();

  return (
    <form onSubmit={handleSubmit(data => console.log(data))}>
      <input {...register('email', { required: '請輸入 email' })} />
      {errors.email && <p>{errors.email.message}</p>}

      <button disabled={isSubmitting}>送出</button>
    </form>
  );
}
```

## 驗證策略

### 前端驗證

適合：

- 必填
- 格式
- 長度

### 後端驗證

一定要有，因為前端驗證無法取代真正安全驗證。

---

## 常見實務問題

- submit 中重複點擊
- 非同步送出中的 loading 狀態
- API error 回填表單
- 動態欄位

---

## 什麼情況要用表單工具

- 欄位多
- 驗證規則多
- 表單邏輯複雜

如果只是簡單搜尋框，不一定需要上完整工具。
