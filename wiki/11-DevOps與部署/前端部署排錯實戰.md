# 前端部署排錯實戰

> 前端部署失敗很多時候不是單一原因，而是 build、環境變數、路由、快取與 API 設定一起交錯造成。

> [[前端工程師]] | [[CI-CD]]

---

## 最常見的三類問題

### 1. 本地能跑，部署後 build 失敗

常見原因：

- 型別錯誤
- lint fail
- 環境變數缺失
- 大小寫路徑問題

### 2. 畫面打得開，但 API 全炸

常見原因：

- `VITE_` 環境變數沒設
- API base URL 錯
- [[CORS]] 設定錯

### 3. 刷新頁面直接 404

如果是 SPA，很可能是主機沒有正確 fallback 到 `index.html`。

---

## 排錯順序

1. 先看 build log
2. 再看環境變數
3. 再看部署平台設定
4. 最後看瀏覽器 console 與 network

---

## 常見案例

### Vite 環境變數沒生效

```ts
const apiUrl = import.meta.env.VITE_API_URL;
```

如果不是 `VITE_` 開頭，前端通常讀不到。

### React Router 刷新 404

部署到靜態主機時，要確認 route fallback。

### 部署成功但舊版資源還在

通常和 [[前端快取策略]] 有關，不一定是部署失敗。

---

## 實務建議

- build 指令要在 CI 先跑過
- 環境變數文件要清楚
- 部署後至少檢查首頁、登入、關鍵 API
